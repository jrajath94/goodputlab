<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| health script to HTTP endpoints | Shell script sends requests to topology ports and parses responses. |
| health script to metrics text | Shell script parses untrusted Prometheus text exposed by containers. |
| health script to sentinel CLI | Shell invokes Python sentinel check and trusts exit code. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-S | Spoofing | model readiness | mitigate | Require `/v1/models` to return `goodputlab-model`, not just HTTP 200. |
| T-01-06-T | Tampering | metrics parsing | mitigate | Match exact vLLM/NIXL metric names and fail if required metrics are absent. |
| T-01-06-R | Repudiation | health result output | mitigate | Print each check stage and `[OK] <topology> healthy` only after all gates pass. |
| T-01-06-I | Information Disclosure | health logs | mitigate | Do not print prompts, tokens, logprobs, or secrets; sentinel logs only pass/fail. |
| T-01-06-D | Denial of Service | repeated health checks | accept | Health is operator-triggered and bounded by curl and sentinel timeouts. |
| T-01-SC | Tampering | shell command execution | mitigate | Use `set -euo pipefail`, quote variables, and avoid eval or sourced environment files. |
</threat_model>
