<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client to topology HTTP ports | OpenAI-compatible requests enter vLLM or proxy containers on localhost-published ports. |
| prefill container to decode container | KV blocks cross process boundary through NIXL UCX. |
| vLLM container to LMCache connector | Disagg-tier writes and reads KV through LMCache PD over NIXL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-S | Spoofing | served model identity | mitigate | Use `--served-model-name goodputlab-model` on every topology and healthcheck `/v1/models` for that exact ID. |
| T-01-03-T | Tampering | NIXL backend selection | mitigate | Encode `backends=["UCX"]` in config JSON and reject any `LIBFABRIC` occurrence in verification. |
| T-01-03-R | Repudiation | topology startup commands | mitigate | Keep all commands in one compose file and Makefile targets so execution logs identify the exact profile. |
| T-01-03-I | Information Disclosure | published metrics | accept | Phase 1 is single-tenant on the dev pod; bind local compose ports and do not include prompt or secret values in configs. |
| T-01-03-D | Denial of Service | GPU memory on one H100 | mitigate | Cap disagg prefill and decode services at `--gpu-memory-utilization 0.45` each. |
| T-01-SC | Tampering | Docker image supply chain | mitigate | Use the pinned vLLM image from Phase 1 research and verify image version in Plan 01-02 before GPU startup. |
</threat_model>
