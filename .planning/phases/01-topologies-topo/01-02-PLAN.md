<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RunPod host to Docker Hub | `provision.sh` pulls the pinned vLLM serving image. |
| RunPod host to Hugging Face | `scripts/pull_model.sh` downloads public model weights into `/workspace/hf`. |
| local shell to GPU runtime | Provisioning invokes Docker with NVIDIA GPU access. |
| provision.sh to colocated endpoint | After boot, provision records a sentinel fixture and runs the post-record health check on port 18000. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-S | Spoofing | model ID resolution | mitigate | Use exact official model IDs from Hugging Face API and print the resolved `MODEL_ID` before startup. |
| T-01-02-T | Tampering | vLLM image and model artifacts | mitigate | Pull a pinned vLLM image, verify `vllm.__version__`, and fail closed if image imports or model probes fail. |
| T-01-02-I | Information Disclosure | shell environment | mitigate | Do not read `.env`, do not print tokens, do not commit runtime env files, and use public model downloads only. |
| T-01-02-D | Denial of Service | RunPod GPU billing and cold start | mitigate | Enforce a 1200-second TOPO-06 budget across image pull + model download + topology boot + sentinel record + post-record health check, print per-stage elapsed time, and fail non-zero when exceeded. |
| T-01-02-R | Repudiation | sentinel fixture provenance | mitigate | Record the sentinel fixture from the trusted colocated topology immediately after boot, before health check, so the post-record check has a fixture to compare against. |
| T-01-SC | Tampering | Docker image and HF model supply chain | mitigate | Use package legitimacy audit from Phase 1 research; use official vLLM image and official Qwen model IDs; no `[SLOP]` packages are introduced. |
</threat_model>
