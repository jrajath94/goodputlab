<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sentinel client to topology endpoint | Sentinel sends deterministic OpenAI-compatible requests to vLLM/proxy endpoints. |
| sentinel fixture to check mode | Local fixture JSON determines expected tokens/logprobs. |
| sentinel daemon to metrics scrape | Daemon exposes `sentinel_drift` over HTTP for Prometheus-compatible scrapes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-T | Tampering | sentinel fixture JSON | mitigate | Fixture filename includes model/version/prompt hash; check mode fails on malformed or missing fields. |
| T-01-05-R | Repudiation | sentinel result logs | mitigate | Print explicit `SENTINEL PASS` or `SENTINEL FAIL` and daemon logs timestamp plus endpoint. |
| T-01-05-I | Information Disclosure | prompt/token logging | mitigate | Use fixed non-sensitive prompt and do not log token arrays or logprobs in daemon output. |
| T-01-05-D | Denial of Service | daemon loop | mitigate | Default interval is 60 seconds and requests use bounded timeouts. |
| T-01-SC | Tampering | Python dependencies | mitigate | Use requests and prometheus-client declared in pyproject and covered by project dependency audit before install. |
</threat_model>
