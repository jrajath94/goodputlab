<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client to proxy | Untrusted OpenAI-compatible JSON enters FastAPI. |
| proxy to prefill/decode | Proxy forwards requests to internal vLLM services. |
| proxy to metrics consumers | Proxy exposes combined metrics for health checks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-S | Spoofing | `X-Request-Id` propagation | mitigate | Generate or propagate a request ID so prefill and decode logs can be correlated. |
| T-01-04-T | Tampering | prefill request body | mitigate | Clone request JSON and cap tokens only in the prefill copy; do not mutate the decode body. |
| T-01-04-R | Repudiation | proxy error handling | mitigate | Preserve upstream status codes and include upstream stage name in logged errors. |
| T-01-04-I | Information Disclosure | `/metrics` response | accept | Phase 1 is single-tenant; metrics include counters/histograms only and no prompt text. |
| T-01-04-D | Denial of Service | proxy upstream calls | mitigate | Use async clients, upstream timeouts appropriate for model serving, and non-success error propagation rather than infinite retries. |
| T-01-SC | Tampering | Python proxy dependencies | mitigate | Use FastAPI and httpx already present in the vLLM image or declared in pyproject; no unreviewed package is introduced by this plan. |
</threat_model>
