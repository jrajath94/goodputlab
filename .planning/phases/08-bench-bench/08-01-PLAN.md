# 08-01: Bench orchestrator + topology campaign harness

## Goal

An offline `BenchOrchestrator` that runs a synthetic trace through the
loadgen → router → mock-vLLM pipeline and emits a `CampaignReport`
per topology.  Used for end-to-end smoke tests of the whole stack;
the real RunPod-backed bench is gated behind approval (cost plan in
RUNPOD.md).

## Files

| File | Purpose |
|------|---------|
| `bench/orchestrator.py` | `BenchOrchestrator`, `Topology`, `CampaignReport` |
| `bench/mock_vllm.py` | `MockVllmServer` — FastAPI app emitting fake tokens |
| `tests/test_orchestrator.py` | End-to-end: trace → telemetry → report |
| `tests/test_mock_vllm.py` | Mock vLLM contract |

## Contract

```python
class Topology(StrEnum):
    COLOCATED = "colocated"
    CHUNKED = "chunked"
    DISAGG = "disagg"
    DISAGG_TIER = "disagg_tier"

class CampaignReport(BaseModel):
    topology: Topology
    n_requests: int
    success_rate: float
    mean_ttft_ms: float
    p95_ttft_ms: float
    cache_hit_rate: float
    reconcile_passes: bool
    notes: list[str]   # "[DRY-RUN]" tag if not real measurements

class BenchOrchestrator:
    def __init__(self, mock_vllm: MockVllmServer): ...
    def run_topology(
        self,
        topology: Topology,
        trace: Trace,
        router: Router,
    ) -> CampaignReport: ...
    def run_campaign(
        self,
        traces: dict[Topology, Trace],
        router_factory: Callable[[Topology], Router],
    ) -> list[CampaignReport]: ...
```

## Dry-run discipline

Every `CampaignReport.notes` starts with `"[DRY-RUN]"` — the report
asserts the orchestrator **ran**, but makes no claim about real
numbers.  The real bench (RunPod H100 + live vLLM) replaces the mock
behind a flag.
