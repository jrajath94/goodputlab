# 04-01: Router A/B verification — cold vs warm regime

## Goal

A `RouterBenchmark` that runs two request streams through the router
and produces a regime-comparison report:

- **Cold regime**: each request has a unique prompt prefix → cache hit rate near 0
- **Warm regime**: many requests share the same prefix → cache hit rate ≥ 0.5

The benchmark proves the LRU prefix cache is doing useful work
(RTR-04 acceptance criterion).

## Files

| File | Purpose |
|------|---------|
| `bench/router_bench.py` | `RouterBenchmark.run_cold()`, `run_warm()`, `RegimeReport` |
| `tests/test_router_bench.py` | Synthetic prompts; assert warm > cold cache hit rate |

## Contract

```python
class RegimeReport(BaseModel):
    n_requests: int
    cache_hits: int
    cache_hit_rate: float
    pools_used: dict[Pool, int]
    decisions: list[PoolDecision]

class RouterBenchmark:
    def __init__(self, router: Router) -> None: ...

    def run_cold(
        self,
        n_requests: int = 100,
        prompt_token_target: int = 200,
        seed: int = 1,
    ) -> RegimeReport: ...

    def run_warm(
        self,
        n_requests: int = 100,
        n_distinct_prefixes: int = 10,
        prompt_token_target: int = 200,
        seed: int = 1,
    ) -> RegimeReport: ...
```

Cold: each request's prompt_text is a fresh `synth_paragraph` — unique.
Warm: pick from `n_distinct_prefixes` shared prefixes; each request
appends a short query-specific suffix.

## Tests

- `test_cold_regime_has_zero_cache_hits` — every prompt unique
- `test_warm_regime_has_high_cache_hit_rate` — 90/100 reuse same 10 prefixes
- `test_warm_uses_fewer_pools_than_cold` — cache sticky
- `test_router_benchmark_decisions_are_serializable`
- `test_router_benchmark_seed_determinism` — same seed → same decisions

This is pure offline measurement; no vLLM, no network.
