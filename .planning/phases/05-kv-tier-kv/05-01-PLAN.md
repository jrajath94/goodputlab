# 05-01: LMCache KV tier abstraction

## Goal

A `LmcacheClient` abstraction that lets the rest of the codebase talk
to LMCache without binding to a specific HTTP API.  In production this
calls the real LMCache sidecar; in tests we plug in a deterministic
mock with controllable hit-rate + eviction behavior.

A `TierPolicy` decides when the router should route a request to the
TIER pool vs. raw PREFILL, based on observed hit rate.

## Files

| File | Purpose |
|------|---------|
| `kv/lmcache_client.py` | `KvStats`, `LmcacheClient` (protocol), `MockLmcacheClient` |
| `kv/tier_policy.py` | `TierPolicy` — hit-rate threshold, capacity-aware |
| `tests/test_kv.py` | Mock-driven hit/miss/eviction; policy thresholds |

## Contract

```python
class KvStats(BaseModel):
    tier: str
    hit_rate: float          # 0..1
    hits: int
    misses: int
    capacity_bytes: int
    used_bytes: int
    eviction_count: int

class LmcacheClient(Protocol):
    def lookup(self, prefix_hash: str) -> bool: ...
    def stats(self) -> KvStats: ...
    def evict(self, prefix_hash: str) -> None: ...

class MockLmcacheClient:
    """Deterministic mock: store up to ``capacity`` entries; LRU evict."""

    def __init__(self, capacity: int = 100, hit_probability: float = 0.9): ...
    def lookup(self, prefix_hash: str) -> bool: ...
    def stats(self) -> KvStats: ...

class TierPolicy:
    """Decide: route this request to TIER pool, or fall through to PREFILL?"""

    def __init__(self, min_hit_rate: float = 0.5, min_capacity_free_pct: float = 10.0): ...
    def should_use_tier(self, stats: KvStats, expected_hit_prob: float) -> bool: ...
```

## Policy logic

1. If `stats.used_bytes / capacity_bytes > (1 - min_capacity_free_pct/100)` → reject (no room)
2. If `stats.hit_rate < min_hit_rate` and `expected_hit_prob < min_hit_rate` → reject (cold tier)
3. Otherwise → use tier

`expected_hit_prob` is the workload-prior (RAG=0.85, chat=0.3) — the
caller passes this in based on the request's workload type.
