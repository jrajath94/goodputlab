# 06-01: EAGLE-3 speculative decoding abstraction

## Goal

A `SpecDecoder` simulator that produces draft-verify outcomes deterministically
(configurable acceptance rate), tracks `goodputlab_spec_proposed_total` /
`goodputlab_spec_acceptance_total`, and **auto-disables** when acceptance
falls below a threshold (SPEC-02).  Honors the **P3 addendum** gate:
speculative decoding is **disabled** in pure disagg (P→D) topologies
because the prefill amortization that makes speculation worthwhile is
absent (SPEC-01).

## Contract

```python
class DraftProposal(BaseModel):
    draft_tokens: list[str]
    proposer: Literal["eagle3", "eagle2", "medusa"] = "eagle3"

class VerifyOutcome(BaseModel):
    accepted_tokens: list[str]
    proposed: int
    accepted: int

class SpecPolicy(BaseModel):
    enabled: bool = True
    min_acceptance_rate: float = 0.4     # auto-disable below this
    min_window: int = 20                # need this many samples before deciding
    # P3 gate: speculative decoding is incompatible with pure disagg.
    topology: Literal["colocated", "chunked", "disagg", "disagg_tier"] = "colocated"

class SpecDecoder:
    def __init__(self, policy: SpecPolicy, acceptance_rate: float = 0.8): ...
    def propose_and_verify(self, n_draft: int = 5) -> VerifyOutcome: ...
    @property
    def is_enabled(self) -> bool: ...
    @property
    def observed_acceptance_rate(self) -> float: ...
```

The simulator uses `random.Random(seed)` for reproducibility. Each
draft token is independently accepted with `acceptance_rate`
probability.

## Auto-disable logic

Maintain a sliding window of the last `min_window` outcomes. When the
window's mean acceptance rate drops below `min_acceptance_rate`, the
decoder flips `enabled=False` and stays off. No auto-recovery in this
plan — that's SPEC-03 territory.
