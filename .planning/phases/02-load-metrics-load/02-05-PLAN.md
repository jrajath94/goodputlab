# 02-05: Prometheus exporter

## Goal

A `MetricsRegistry` + FastAPI `/metrics` endpoint that exposes the
counters/gauges/histograms needed to plot goodput curves and detect
failure modes (OBS-01). Grafana scrapes `/metrics`; the bench harness
also reads it back for the final report.

## Files

| File | Purpose |
|------|---------|
| `obs/registry.py` | `MetricsRegistry` — typed counters/gauges/histograms |
| `obs/server.py` | FastAPI app with `/metrics` endpoint |
| `tests/test_registry.py` | Increment → scrape → text contains expected names |
| `tests/test_obs_server.py` | GET /metrics returns text/plain prometheus format |

## Metric inventory

| Metric | Type | Labels | Source |
|--------|------|--------|--------|
| `goodputlab_ttft_ms` | Histogram | slo_class | `RequestTelemetry.ttft_ms` |
| `goodputlab_itl_ms` | Histogram | slo_class | derived from `per_token_ts_ns` |
| `goodputlab_queue_depth` | Gauge | pool | router (Phase 3) |
| `goodputlab_kv_tier_hit_total` | Counter | tier | LMCache hit rate (Phase 5) |
| `goodputlab_spec_acceptance_total` | Counter | — | EAGLE-3 accepted tokens (Phase 6) |
| `goodputlab_spec_proposed_total` | Counter | — | total draft tokens proposed |
| `goodputlab_role_flip_total` | Counter | from, to | router role flip (P/D swap, Phase 3) |
| `goodputlab_request_total` | Counter | status_code | every send_one result |
| `goodputlab_request_error_total` | Counter | error_class | every non-200 |

`prometheus-client` is already in `pyproject.toml` deps. We use
`CollectorRegistry` (one per MetricsRegistry instance) so tests are isolated.

## Test strategy

- `test_registry_records_ttft` — observe(histogram_value) → text contains bucket
- `test_registry_records_role_flip` — increment_counter → text contains counter
- `test_obs_server_metrics_endpoint_returns_prometheus_text` — GET → 200, content-type
- `test_obs_server_endpoint_isolates_registries` — two registries → independent state
- `test_obs_server_handles_empty_registry` — fresh registry → valid empty text
