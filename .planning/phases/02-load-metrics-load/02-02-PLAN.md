# 02-02: Per-workload trace generators

## Goal

Three pluggable trace generators — `ChatTraceGenerator`, `RagTraceGenerator`,
`AgenticTraceGenerator` — that produce a `Trace` (list of `RequestSpec`) for
the three workload shapes defined in REQUIREMENTS.md:

- **CHAT** (LOAD-01): multi-turn, shared system prompt, 0.5–2K input tokens,
  50–500 output tokens, seedable
- **RAG** (LOAD-02): 8–32K input tokens (long context with high prefix
  overlap), short output, ~80% shared prefix
- **AGENTIC** (LOAD-03): bursty, high prefix overlap (shared system + tool
  defs + accumulated history)

All three use `random.Random(seed)` exclusively — LOAD-07 contract.

## Why this is its own plan

02-01 established the *types*. 02-02 establishes the *content shapes* the
router and metrics reconciler will see.  Generator output flows directly
into 02-03 (replay engine) and 02-04 (reconciler) — without realistic
prompt shapes here, those plans are testing nothing.

## Generator contract

Each generator exposes:
```python
class XxxTraceGenerator:
    def __init__(self, config: XxxWorkloadConfig) -> None: ...
    def generate(self) -> Trace: ...
```

Where `XxxWorkloadConfig` carries `n_requests`, `seed`, workload-specific
ranges, and any shared prefix (system prompt, document corpus, tool defs).

The returned `Trace` carries the workload's `WorkloadType`, the seed,
the arrival config (Poisson at `rate_per_sec` from config), and the list
of `RequestSpec` — order is preserved for index-paired arrival.

## Synth text (deterministic)

Real corpora are out of scope.  `loadgen/synth_text.py` provides a
deterministic paragraph builder:

```python
def synth_paragraph(rng: random.Random, target_tokens: int) -> str: ...
def synth_document(rng: random.Random, target_tokens: int) -> str: ...
def synth_token_count(text: str) -> int:
    """Cheap whitespace-tokenizer; used by all generators for token budgets."""
```

The word list ships inline in the module (~200 generic English words) so
the generators produce ~plausible English-shaped prompts without
shipping a corpus.  Different seeds produce visibly different text; the
same seed produces byte-identical text.

## Workload shapes

### Chat (LOAD-01)

- Shared `system_prompt` (~200 tokens, same for every request)
- Each request: 1–N user turns (configurable via `n_turns_range`); each
  turn is 50–500 tokens
- Output: 50–500 tokens per request
- Arrival: Poisson at `rate_per_sec` from config
- Default: `prompt_tokens ∈ [500, 2000]`, `output_tokens ∈ [50, 500]`

### RAG (LOAD-02)

- Shared system prompt (~150 tokens: "You are a helpful assistant...")
- Shared "retrieved context": a corpus of K documents, each 1–4K tokens
- Each request: include a configurable fraction (default 0.8 = 80%) of
  the corpus + a unique user query (50–200 tokens)
- Output: 50–200 tokens (short answer)
- Default: `prompt_tokens ∈ [8000, 32000]`, `output_tokens ∈ [50, 200]`
- Asserted: ≥75% of any two requests' prompt text is byte-identical
  (the "shared prefix" property)

### Agentic (LOAD-03)

- Shared system prompt + tool definitions (~500 tokens)
- Each request represents one tool call in a long agentic loop
- Prefix includes: system + tool defs + accumulated history
- User "observation" varies per call (50–500 tokens)
- Output: 100–1000 tokens (the agent's reasoning + next action)
- Default: `prompt_tokens ∈ [2000, 16000]`, `output_tokens ∈ [100, 1000]`
- Asserted: ≥60% of any two requests' prompt text is byte-identical
  (prefix grows monotonically — late calls have everything early calls had)

## Tests

### `tests/test_synth_text.py`

1. `test_synth_paragraph_byte_identical` — same RNG state → same text
2. `test_synth_paragraph_target_tokens_within_15pct` — actual token count within 15% of target
3. `test_synth_document_grows_with_target` — doubling target roughly doubles size
4. `test_synth_token_count_cheap_whitespace` — count is non-negative and matches `len(text.split())`

### `tests/test_chat_generator.py`

1. `test_chat_trace_has_n_requests` — config.n_requests respected
2. `test_chat_byte_identical` — two `generate()` calls → identical Trace JSON
3. `test_chat_prompt_tokens_in_range` — every request's prompt_tokens within config bounds
4. `test_chat_output_tokens_in_range` — every request's output_tokens within config bounds
5. `test_chat_shared_system_prompt` — every request's prompt_text starts with system_prompt

### `tests/test_rag_generator.py`

1. `test_rag_trace_has_n_requests`
2. `test_rag_byte_identical`
3. `test_rag_prompt_tokens_at_least_8k` — by default
4. `test_rag_prefix_overlap_at_least_75pct` — pairwise byte overlap of any two prompts ≥ 0.75
5. `test_rag_short_output` — output_tokens ≤ 200 (LOAD-02)

### `tests/test_agentic_generator.py`

1. `test_agentic_trace_has_n_requests`
2. `test_agentic_byte_identical`
3. `test_agentic_prefix_grows_monotonically` — later requests have ≥ as much shared prefix as earlier (history accumulates)
4. `test_agentic_prefix_overlap_at_least_60pct` — pairwise overlap
5. `test_agentic_output_tokens_in_range`
