# 03-01: Router — cache-aware prefix routing + admission control

## Goal

A `Router` that picks a `Pool` (prefill / decode / colocated / tier) for
every `RequestSpec` based on (a) the request's SLO class, (b) the
prefix-hash → pool cache, (c) current pool queue depth and an
admission policy. Decisions are deterministic and observable
(recorded per-request so Phase 8 can plot role-flip rates).

## Why this is its own plan

The router is the Tier-1 differentiator — SOTA inference projects
demote routers to "load balancer" status. Here it's the whole control
plane. Three sub-pieces, all tight enough to ship in one plan:

1. **Pool model** — typed `Pool` enum + `PoolState` snapshot
2. **Prefix cache** — `lru_cache(maxsize=N)` keyed by `sha256(prompt_text)` (or prefix of it)
3. **Admission policy** — token-bucket-ish guard: reject when pool depth > capacity × headroom

## Router contract

```python
class Pool(StrEnum):
    PREFILL = "prefill"
    DECODE = "decode"
    COLOCATED = "colocated"
    TIER = "tier"

class PoolState(BaseModel):
    pool: Pool
    queue_depth: int = 0
    capacity: int = 64

class PoolDecision(BaseModel):
    pool: Pool
    admitted: bool
    reason: str  # "cache_hit" | "load_balance" | "admission_reject" | ...
    prefix_hash: str | None = None  # cache key used

class AdmissionPolicy(BaseModel):
    headroom_pct: float = 0.8   # reject when depth > capacity * headroom
    interactive_bypass: bool = True  # interactive SLO bypasses admission

class Router:
    def __init__(
        self,
        policy: AdmissionPolicy,
        prefix_cache_size: int = 1024,
    ) -> None: ...

    def register_pool(self, state: PoolState) -> None: ...
    def update_depth(self, pool: Pool, depth: int) -> None: ...
    def route(self, spec: RequestSpec) -> PoolDecision: ...
```

## Routing rules

1. `spec.slo_class == INTERACTIVE` and any pool under headroom → pick the pool with the **cache hit** for the spec's prefix; tiebreak by lowest queue depth.
2. `spec.slo_class == INTERACTIVE` and pools at headroom → **reject** unless `interactive_bypass=True`.
3. `spec.slo_class == BATCH` → prefer lowest-depth pool regardless of cache.
4. `spec.workload == RAG` and a `tier` pool is registered with capacity → prefer `tier` (KV reuse beats raw prefill).
5. Cache key = first 256 chars of `spec.prompt_text` (stable, fast, ~64 tokens).

## Observability

Every `route()` call returns a `PoolDecision`; the loadgen records it
on `RequestTelemetry.routed_pool`. Role-flip is detected downstream
in `obs/registry.MetricsRegistry.inc_role_flip(...)` when the same
prefix hashes to a different pool across consecutive runs.
